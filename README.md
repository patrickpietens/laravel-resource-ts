Transform your Laravel API Resources into TypeScript type definitions. The package uses a hybrid approach: static analysis of your `toArray()` methods handles common patterns automatically, while PHP 8 attributes let you take over for anything the analyzer can't infer.

## Installation

```bash
composer require patrickpietens/laravel-resource-ts --dev
```

Publish the config file:

```bash
php artisan vendor:publish --tag="resource-ts-config"
```

## Quick start

Given a typical API Resource:

```php
class UserResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'name' => $this->name,
            'email' => $this->email,
            'is_admin' => $this->is_admin,
            'posts' => PostResource::collection($this->posts),
            'profile' => $this->whenLoaded('profile', fn () => ProfileResource::make($this->profile)),
        ];
    }
}
```

Run the generator:

```bash
php artisan typescript:generate
```

Output (`resources/js/types/resources.d.ts`):

```typescript
// This file is auto-generated by laravel-resource-ts.
// Do not edit manually.

export type User = {
  id: number;
  name: string;
  email: string;
  is_admin: boolean;
  posts: Post[];
  profile?: Profile;
};
```

## How it works

The package uses two strategies to determine TypeScript types, applied in order of precedence:

### 1. Static analysis (automatic)

The analyzer parses your `toArray()` method's AST and infers types from:

| PHP pattern | TypeScript result |
|---|---|
| `(int) $this->field` | `number` |
| `(string) $this->field` | `string` |
| `(bool) $this->field` | `boolean` |
| `(float) $this->field` | `number` |
| `'hello'` (string literal) | `string` |
| `42` (int literal) | `number` |
| `true` / `false` | `boolean` |
| `null` | `null` |
| `SomeResource::make(...)` | `Some` (nested type) |
| `SomeResource::collection(...)` | `Some[]` (array of nested type) |
| `$this->when(...)` | optional field (`?`) |
| `$this->whenLoaded(...)` | optional field (`?`) |
| `$this->whenNotNull(...)` | optional field (`?`) |
| `$this->field` (with model casts) | inferred from `$casts` |

### 2. Model cast inference

When you specify the underlying model (via attribute or `@mixin` docblock), the package reads the model's `$casts` array to infer types for `$this->property` accesses:

| Eloquent cast | TypeScript type |
|---|---|
| `integer`, `int` | `number` |
| `float`, `double`, `decimal` | `number` |
| `string` | `string` |
| `boolean`, `bool` | `boolean` |
| `date`, `datetime` | `string` |
| `timestamp` | `number` |
| `array`, `collection` | `unknown[]` |
| `object` | `Record<string, unknown>` |

### 3. Attribute overrides (explicit)

When static analysis can't determine a type (or gets it wrong), use the `#[Typescript]` attribute:

```php
use ResourceTs\Attributes\Typescript;

#[Typescript(
    name: 'User',                                       // custom TS type name
    model: \App\Models\User::class,                     // model for cast inference
    overrides: ['metadata' => 'Record<string, string>'], // override specific fields
    exclude: ['internal_field'],                         // exclude fields from output
)]
class UserResource extends JsonResource
{
    // ...
}
```

All attribute parameters are optional. You can use any combination:

```php
// Just specify the model for cast inference
#[Typescript(model: User::class)]
class UserResource extends JsonResource { ... }

// Just override a tricky field
#[Typescript(overrides: ['config' => '{ theme: string; locale: string }'])]
class SettingsResource extends JsonResource { ... }

// Just rename the output type
#[Typescript(name: 'AuthUser')]
class AuthenticatedUserResource extends JsonResource { ... }
```

Alternatively, you can use a `@mixin` docblock to specify the model:

```php
/** @mixin \App\Models\User */
class UserResource extends JsonResource { ... }
```

## Configuration

```php
return [
    // Where to write the generated .d.ts file
    'output' => resource_path('js/types/resources.d.ts'),

    // Directories to scan for Resource classes
    'paths' => [
        app_path('Http/Resources'),
    ],

    // Use model $casts for type inference
    'auto_discover_models' => true,

    // Strip "Resource" suffix from type names (UserResource -> User)
    'strip_resource_suffix' => true,

    // Sort types alphabetically in output
    'sort_types' => true,

    // Add prefix/suffix to all type names
    'type_prefix' => '',
    'type_suffix' => '',
];
```

## Command options

```bash
# Generate all resources
php artisan typescript:generate

# Generate for a specific resource
php artisan typescript:generate --resource="App\Http\Resources\UserResource"

# Write to a custom output path
php artisan typescript:generate --output="resources/js/types/api.ts"
```
